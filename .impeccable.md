## Design Context

### Users
Layer1 is a local editorial workbench for long sessions of quote curation. The user reads txt files locally in the browser, reviews extracted candidates in batches, commits approved entries, and revisits the stored quote library and almanac history.

### Brand Personality
Restrained, trustworthy, calm.

The interface should feel stable and focused rather than expressive or promotional. It is a tool for sustained judgment work, not a showcase.

### Aesthetic Direction
Use the reference images in `docs/提取配置.png`, `docs/待审核清单.png`, `docs/语料名句库.png`, and `docs/语料宜忌.png` as the visual baseline.

Direction:
- Light mode only
- Warm white or soft beige background
- Thin warm-gray borders
- Large radii
- Compact but breathable density
- Low-saturation tags and status accents
- Lists over heavy admin tables

Anti-direction:
- Do not look like a generic SaaS dashboard
- Do not look like a marketing page
- Do not look like a note-taking app
- Do not trade review efficiency for decoration

### Design Principles
- Use clear language
- Limit distractions
- Group related infomation
- Avoid unnecessary details
- Emphasize important content

Additional implementation guardrails:
- Re-read `docs/architecture.md` before major Layer1 changes
- Reference mocks are visual guidance only; business fields come from the architecture doc
- txt intake is local file reading, not a server upload API
- Do not introduce `epub`, `weight`, or the mistaken `改` action into Layer1
